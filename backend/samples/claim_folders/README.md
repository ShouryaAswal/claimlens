# Sample claim folders

Nine synthetic claim folders (3 per LOB) for exercising the full pipeline end-to-end, generated by `scripts/generate_sample_claims.py`. Upload any one of these folders whole (via the "Browse folder" button on the Start Claim page) to test folder upload with nested subdirectories.

| Folder | LOB | Scenario |
|---|---|---|
| `auto_complete_clean/` | Auto | All mandatory docs present, fields agree -- expect STP candidate |
| `auto_incomplete/` | Auto | Missing police report + policy declaration -- expect needs_review/high_risk |
| `auto_conflicting_values/` | Auto | Two repair estimates disagree on dollar amount; includes a 22-page supplemental PDF |
| `property_complete_clean/` | Property | All mandatory docs present, fields agree -- expect STP candidate |
| `property_incomplete/` | Property | Missing inspection report, receipts, policy declaration |
| `property_conflicting_values/` | Property | Initial vs. revised contractor estimate disagree on loss amount |
| `health_complete_clean/` | Health | All mandatory docs present, fields agree -- expect STP candidate |
| `health_incomplete/` | Health | Missing discharge summary + itemized bill |
| `health_conflicting_values/` | Health | Original vs. corrected itemized bill disagree on billed amount |

Every folder also includes at least one irrelevant file under `irrelevant/` -- either off-topic but readable (an all-hands notes DOCX) or an unsupported extension (a stray .csv) -- to confirm the pipeline classifies/ignores noise rather than getting confused by it.
